# Daily Task Manager

## Game Overview
Daily Task Manager is a mobile productivity application — not a game — built for individuals who need a clean, focused way to manage personal tasks. The core loop is simple: create a task, complete it, clear it. What makes it useful is its frictionless design: no accounts required to start, no bloat, just task management that gets out of the user's way.

> **Note:** This is a productivity app, not a game. The document structure has been adapted accordingly while preserving the requested format. Game-specific sections that do not apply are marked **[N/A — Productivity App]**.

---

## Problem / Opportunity
Most task apps are either too complex (Notion, Jira) or buried inside ecosystems users don't want (Apple Reminders, Google Tasks). There is clear demand for a standalone, no-friction mobile to-do app that works immediately without onboarding overhead. Target users are individuals managing personal daily workloads who want speed and simplicity over features.

---

## Target Players

- **The Busy Individual**: Ages 18–45, light-to-moderate app user, manages personal errands, work tasks, and daily chores. Wants to capture tasks quickly and feel a sense of completion when done. No tolerance for complex setup.
- **The Productivity Minimalist**: Ages 25–40, intentionally avoids feature-heavy apps, prefers focused single-purpose tools. Wants a clean list, nothing more. Values speed of entry and visual clarity above all else.

## Game Design

### Core Gameplay Loop
*[Adapted: Core User Flow]*

1. User opens the app — lands directly on the task list (no login screen on first open)
2. User taps **"Add Task"** — enters a title and optionally sets a due date
3. Task appears in the **Incomplete** list, sorted by due date ascending [Assumed: default sort order]
4. User taps a task to **mark it complete** — task moves to the Completed filter view
5. User can **delete** any task via swipe-to-delete or a delete button [Assumed: swipe gesture on mobile]
6. User filters between **All / Incomplete / Complete** views to manage their list
7. Loop repeats daily — user returns, adds new tasks, clears completed ones

---

### Game Mechanics
*[Adapted: App Mechanics]*

- **Create Task**: Tap "+" or "Add Task" CTA → modal or bottom sheet appears with two fields: `Title` (text, required, max 120 characters [Assumed]) and `Due Date` (date picker, optional). Confirm tap saves task to list.
- **Complete Task**: Tap checkbox or task row to toggle complete state. Visual change: strikethrough on title, checkbox fills, task re-categorised. Toggle is reversible — tapping again marks incomplete.
- **Delete Task**: Swipe left on task row to reveal red Delete button [Assumed: iOS-style swipe action; Android: long-press context menu or swipe]. Tap Delete → task permanently removed with no undo in v1 [Assumed: no undo toast in v1].
- **View All Tasks**: Default list view shows all tasks regardless of status, sorted by due date ascending, with incomplete tasks shown above complete ones [Assumed: secondary sort grouping].
- **Filter Tasks**: Filter tabs or segmented control at top of list — three states: **All**, **Incomplete**, **Complete**. Switching filter is instant, no network call required (local data).

## Features by Area

### Core Functionality
- Create task with required `Title` field (text input, 120 char max [Assumed]) and optional `Due Date` field (date picker, no time component in v1 [Assumed])
- Save task to local persistent storage on confirm
- Display all tasks in a scrollable vertical list
- Show task title, due date (if set), and completion checkbox per row
- Mark task complete/incomplete via checkbox toggle — persists state immediately
- Visually distinguish complete tasks (strikethrough, muted colour [Assumed])
- Visually flag overdue tasks — due date in red if date has passed and task is incomplete [Assumed]
- Delete task permanently via swipe gesture (iOS) / long-press or swipe (Android)
- Filter list by: All / Incomplete / Complete — instant local filter, no reload

### UI / UX
- **Task List Screen**: Primary screen. Contains filter control, scrollable task list, and Add Task FAB or button. Shows empty state when no tasks match current filter.
- **Add Task Screen / Modal**: Bottom sheet or modal overlay. Fields: Title (text input, autofocused on open [Assumed]), Due Date (optional, date picker). CTA: Save / Add. Secondary: Cancel / Dismiss.
- **Empty State Views**: Friendly message + illustration for (a) no tasks at all, and (b) no tasks matching current filter.
- **Delete Confirmation**: Swipe-to-delete reveals red Delete button — single tap deletes immediately. No confirmation dialog in v1 [Assumed: keep friction low].
- Navigation: Single-screen app — no tab bar or navigation stack required in v1 [Assumed].

### Backend & Services
- **Local Data Persistence**: All task data stored locally on device. Survives app close and device restart.
- **Data Schema — Task Object**:
  - `id`: UUID (auto-generated)
  - `title`: String (required)
  - `dueDate`: Date (optional, nullable)
  - `isComplete`: Boolean (default: false)
  - `createdAt`: Timestamp (auto-set on creation)
- **No backend, no auth, no API calls in v1.**

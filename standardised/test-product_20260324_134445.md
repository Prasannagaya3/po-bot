# Simple Todo App

## App Overview
Simple Todo is a lightweight mobile task management app for individuals who need a no-friction way to capture, track, and complete daily tasks. The core loop is simple: create a task, work through your list, mark items done. Its value lies in its simplicity — no bloat, no social features, no noise.

## Problem / Opportunity
Most todo apps are over-engineered with collaboration tools, AI features, and subscription paywalls that alienate users who just need a clean personal task list. Simple Todo fills the gap for users who want a fast, reliable, offline-capable task manager with zero learning curve. The target user is a solo individual managing their own daily workload.

## Target Players

- **Busy Individual (Primary)**: Ages 18–45, light-to-moderate smartphone user, wants to quickly capture tasks and check them off without navigating complex UI or setting up accounts
- **Productivity Minimalist (Secondary)**: Ages 25–55, deliberately avoids feature-heavy apps, wants a distraction-free list experience with no upsells or social clutter

## App Design

### Core User Loop
1. User opens the app — lands directly on the **All Tasks** list view
2. User taps **"+ Add Task"** — enters a title and optional due date
3. Task appears in the list immediately
4. User works through their day, tapping tasks to **mark as complete**
5. User optionally filters list by **Complete / Incomplete** to focus on remaining work
6. User deletes tasks they no longer need via swipe-to-delete or long-press action
7. Loop repeats each session — list persists between app launches

### App Mechanics
- **Create Task**: Tap "+ Add Task" button → input sheet appears with a text field (title, required) and a date picker (due date, optional) → confirm saves task immediately to local list
- **Mark Complete**: Tap checkbox or task row to toggle complete state — completed tasks show a strikethrough on the title and move to "Completed" visual grouping [Assumed: visual strikethrough on completion]
- **Delete Task**: Swipe left on any task row to reveal a red "Delete" action — confirm deletion removes task permanently with no undo [Assumed: no undo/restore in v1]
- **View All Tasks**: Default list view shows all tasks sorted by due date ascending, with no-due-date tasks at the bottom [Assumed: default sort order is due date ascending]
- **Filter Tasks**: Toggle tabs or segmented control at the top of the list — options are "All", "Incomplete", "Complete" — filter persists during session but resets to "All" on next app launch [Assumed: filter resets on relaunch]

## Features by Area

### Core App Functionality
- Create task with a required **title** (text input, max 200 characters) [Assumed: character limit] and optional **due date** (date only, no time) selected via native date picker
- Mark any task as **complete** by tapping its checkbox — toggleable back to incomplete
- **Delete** any task via swipe-left gesture with a destructive confirmation action
- Display all tasks in a **scrollable list view**, sorted by due date ascending, no-due-date tasks at the end
- **Filter** the task list by: All Tasks / Incomplete / Complete using a segmented control or tab bar
- **Persist all tasks locally** across app sessions — data survives app close and device restart

### UI / UX
- **Launch → List**: App opens directly to task list (no onboarding, no login screen)

- **Empty State — All Complete**: Positive confirmation message when all tasks are ticked off
- **Filter Tabs**: Persistent segmented control above the list — "All / Incomplete / Complete"
- **Task Row Design**: Checkbox | Task Title | Due Date label (right-aligned) — completed rows show strikethrough title and greyed styling

## Open Questions

2. **Monetisation strategy**: Is this a free internal tool, a free public app, or will a paid/freemium model be added post-v1? This affects App Store category and metadata.
3. **Data loss on delete**: Is swipe-to-delete with no undo acceptable, or do we need a soft delete / trash bin even in v1?
4. **Due date display**: How should overdue tasks be displayed? Red text, warning icon, or no visual distinction? [Not specified in input]
5. **Sort order**: Is due-date-ascending the correct default? Should incomplete tasks always appear above completed tasks regardless of date?
6. **Filter persistence**: Should the selected filter (All/Incomplete/Complete) be remembered between app sessions, or always reset to "All" on launch?
7. **Edit task**: Can a user edit a task's title or due date after creation? This is not listed as a feature but will almost certainly be needed — confirm if in or out of scope for v1.
8. **Analytics / Crash reporting**: Should we integrate any lightweight crash reporting (e.g. Firebase Crashlytics) even if full analytics are out of scope?
9. **App Store accounts**: Are Apple Developer Program and Google Play Console accounts already set up and accessible?

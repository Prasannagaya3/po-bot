# Simple Todo App

## Game Overview
> **⚠️ Note:** The submitted input describes a **productivity mobile application**, not a game. This document has been reformatted as a clean mobile app product specification while preserving the requested structure as closely as possible. Game-specific sections have been adapted to fit the product domain.

Simple Todo App is a lightweight mobile task management application for individuals who need a friction-free way to capture, organise, and complete daily tasks. The core loop is simple: create a task, work through it, mark it done, and stay on top of what remains. What makes it useful is its intentional simplicity — no bloat, no team overhead, no noise.

---

## Problem / Opportunity
Most to-do apps are over-engineered for solo users — packed with collaboration tools, complex project hierarchies, and subscription paywalls. There is a clear gap for a clean, zero-friction personal task manager that does exactly what a solo user needs and nothing more. The target user is an individual who wants to capture tasks quickly and track completion without a learning curve.

---

## Target Players

- **Busy Individual (Primary)**: Ages 18–45, moderate smartphone proficiency, wants to capture and track personal daily tasks quickly without setup or onboarding friction
- **Light Productivity User (Secondary)**: Ages 16–60, any experience level, wants a simple checklist-style tool as a lightweight alternative to calendar or notes apps

## Game Design

### Core Gameplay Loop
*(Adapted: Core User Flow)*

1. User opens the app and lands on the **All Tasks** list view
2. User taps **"+ Add Task"** and enters a title and optional due date
3. Task appears in the list under **Incomplete**
4. User works through their day; taps a task to **mark it complete**
5. Completed tasks move to or are accessible via the **Completed filter**
6. User can **delete** any task they no longer need
7. User returns next session and repeats the cycle

---

### Game Mechanics
*(Adapted: App Interactions & Behaviours)*

- **Create Task**: Tap a persistent "+" FAB (Floating Action Button); input modal accepts a plain-text title (required, max 120 characters) and a due date via a native date picker (optional)
- **Mark Complete**: Tap a checkbox or swipe-right gesture on a task row; triggers a visual strikethrough and moves the task to the Completed state — [Assumed: action is immediate with no confirmation dialog]
- **Delete Task**: Swipe-left on a task row to reveal a Delete action; tapping Delete removes the task permanently — [Assumed: a brief undo snackbar (~4 seconds) appears post-deletion to prevent accidental loss]
- **View All Tasks**: Default landing view shows all tasks (incomplete and complete) in a flat chronological list, sorted by due date ascending, with no-due-date tasks at the bottom — [Assumed: sort order]
- **Filter Tasks**: Toggle control at the top of the list switches between three states: **All**, **Incomplete**, **Complete** — filter state persists for the session but resets on app relaunch — [Assumed: session persistence behaviour]

## Features by Area

### Core Functionality
- Create a task with a required title (max 120 chars) and optional due date
- Mark a task as complete (checkbox tap or swipe-right)
- Delete a task (swipe-left with confirm via undo snackbar)
- View all tasks in a chronological list (sorted by due date, nulls last)
- Filter task list by: All / Incomplete / Complete
- Persist all task data locally across app sessions
- Empty state UI for lists with zero results
- Input validation: block task save if title is empty

### UI / UX
- Launch → All Tasks List (default view)
- Task List Screen: filter toggle, task rows with checkbox + title + due date, FAB
- Add Task Modal / Bottom Sheet: title input, date picker, Save and Cancel actions
- Swipe-left: delete action with undo snackbar (4-second timeout)
- Swipe-right: mark complete action — [Assumed]
- Empty state screen: illustration + prompt to add first task
- All interactions must be completable with one hand in portrait mode

### Backend & Services
- Local SQLite database schema:
  - `tasks` table: `id` (UUID), `title` (string), `due_date` (nullable date), `is_complete` (boolean), `created_at` (timestamp), `updated_at` (timestamp)
- CRUD operations: Create, Read, Update (complete toggle), Delete
- No remote backend, no user authentication, no API calls in v1

### Store & Deployment
- iOS: App Store submission — requires Apple Developer account, app icon set, privacy manifest (no data collected in v1 simplifies this)
- Android: Google Play submission — requires Play Console account, store listing assets (screenshots, feature graphic, description)
- Both stores: Privacy policy required even for no-data apps — [Assumed: legal to confirm]
- App Store rating: [Assumed: 4+ / Everyone]

4. **Due date behaviour**: Should overdue tasks (past due date, still incomplete) be visually flagged (e.g. red date text)? Not specified in input.
5. **Sort order**: Should the task list be manually reorderable (drag-to-reorder), or is chronological/due-date sort sufficient for v1?
6. **Filter persistence**: Should the selected filter (All/Incomplete/Complete) persist between app sessions or always reset to "All" on launch?
7. **Complete task behaviour**: Do completed tasks remain visible in the "All" view with a strikethrough, or are they hidden from the default view and only accessible via the "Complete" filter?
8. **Delete confirmation**: Is a swipe-left + undo snackbar sufficient, or does the stakeholder require a confirmation dialog before deletion?
9. **App name & branding**: Is "Simple Todo App" the final product name, or a working title? Brand colours and typography not yet defined.

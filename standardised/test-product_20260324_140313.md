# Simple Todo App

## Game Overview
> ⚠️ **Note:** This input describes a **productivity mobile application**, not a game. The document below has been structured accordingly, adapting the game product template to a software product format. Game-specific sections have been reinterpreted as app equivalents.

Simple Todo App is a lightweight mobile task management application for individuals who need a fast, frictionless way to capture and complete daily tasks. The core loop is: create a task → work through it → mark it complete → review progress. What makes it useful is its intentional simplicity — no clutter, no account required, no learning curve.

---

## Problem / Opportunity
Most task management apps (Todoist, Notion, Things 3) are over-engineered for users who simply need a daily to-do list. There is a clear gap for a zero-friction, single-user task app that opens fast, lets you add a task in under 5 seconds, and stays out of your way. The target user is an individual who wants personal task tracking without the overhead of a full productivity suite.

---

## Target Players

- **Casual Personal User**: Age 18–45, low-to-medium tech savvy, wants to jot down tasks quickly without creating an account or learning a system. Values speed and simplicity above all.
- **Light Professional User**: Age 25–50, moderate tech savvy, uses the app as a quick personal capture tool alongside heavier work tools (Slack, Jira, etc.). Wants reliable task state persistence and clean UI.

## App Design

### Core Usage Loop
1. User opens the app — lands directly on the task list (no loading screen, no login wall)
2. User taps **"Add Task"** — enters a title and optional due date
3. Task appears in the **Incomplete** list
4. User works through their day, tapping tasks to **mark complete**
5. Completed tasks move to (or are filterable in) the **Completed** view
6. User deletes stale or unwanted tasks as needed
7. Loop repeats daily

---

### App Mechanics / Features

#### Task Management
- **Create Task**: User inputs a task title (required, max 200 characters) and an optional due date via a date picker. Task is saved immediately on confirmation.
- **Mark Task as Complete**: Single tap on a checkbox or swipe gesture toggles a task from incomplete → complete. Action is immediate with visual feedback (strikethrough text, colour change). [Assumed: toggle is reversible — tapping again marks incomplete]
- **Delete Task**: Swipe-to-delete gesture (iOS standard) or long-press context menu (Android standard) with a confirmation prompt to prevent accidental deletion. [Assumed: deletion is permanent, no recycle bin in v1]
- **View All Tasks**: Default view shows all tasks in a flat list, ordered by due date ascending, with tasks without a due date appearing last. [Assumed: chronological sort as default]
- **Filter Tasks**: Toggle filter control at the top of the list with two states — **Incomplete** (default on open) and **Complete**. Filter persists during the session. [Assumed: does not persist between app launches — resets to Incomplete on reopen]

#### Due Date Handling
- Due date is optional on all tasks
- Tasks with a due date display it beneath the title in the list
- [Assumed: no push notifications for due dates in v1 — see Out of Scope]
- [Assumed: overdue tasks are visually flagged, e.g. due date text turns red, but no alert is fired]

---

### Progression / State
- No formal progression system — app state is the task list itself
- Completed task count may be displayed as a lightweight summary (e.g. "3 of 7 tasks complete today") [Assumed: optional motivational element, low priority]

---

### Success / Failure States
- **Success state**: All tasks in the Incomplete list are marked complete — empty state screen shown with positive reinforcement message (e.g. "All done for today! ✓")
- **Failure / Empty state**: No tasks exist yet — onboarding empty state with a prompt to add the first task
- **Error state**: Task save fails (e.g. storage error) — inline error message shown, task not dismissed from input form

## Features by Area

### Core App Functionality
- Create task with title (required) and due date (optional)
- Mark task complete / incomplete via checkbox toggle
- Delete task with swipe gesture + confirmation
- Persistent local storage of all task data
- Visual overdue indicator for past-due tasks

### UI / UX
- Task list screen as the default launch screen (no splash/login)
- Incomplete / Complete filter toggle — visible on main screen
- Empty state screens for zero tasks and all-complete states
- Add Task entry point via FAB on main screen
- Inline error handling for save/delete failures
- Responsive layout for small (SE) to large (Pro Max) screen sizes

### Backend & Services
- Local SQLite (or equivalent) database schema:
  - `tasks` table: `id`, `title`, `due_date` (nullable), `is_complete` (boolean), `created_at`, `updated_at`
- No remote backend, no authentication, no API calls in v1

### Store & Deployment
- iOS: App Store submission — requires Apple Developer account, app privacy labels (local data only), screenshot set for 3 device sizes
- Android: Google Play submission — requires Play Console account, content rating questionnaire, adaptive icon asset
